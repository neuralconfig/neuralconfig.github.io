About Page Layout Options

Three different visual approaches for your about page

Layout A: Developer Portfolio

GitHub-inspired, project-focused, minimal business language

Visual Style

  • Pinned repositories grid (featured projects)
  • Contribution-style activity calendar
  • Tech stack badges front and center
  • README-style bio section
  • Clean, developer-first aesthetic

Content Focus

  • "Building" and "Experimenting" language
  • Projects take visual priority
  • Stats: GitHub stars, commits, downloads
  • Learning journey emphasized
Key Features:
  • Project showcase cards with GitHub stats
  • Skills presented as tech badges
  • Timeline of recent activity/builds
  • Minimal text, maximum code/projects
Pros
  • Very developer-authentic
  • Minimal conflict with day job
  • Appeals to AI/tech companies
  • Community-focused
Cons
  • Less professional polish
  • May undersell experience
  • Requires GitHub API integration

Layout B: Personal Tech Blog

Narrative-driven, journey-focused, approachable

Visual Style

  • Personal photo/avatar section
  • Story-driven narrative layout
  • Featured blog posts integration
  • Timeline of learning journey
  • Warm, conversational design

Content Focus

  • "My journey from X to Y" narrative
  • Current learning goals highlighted
  • Projects as case studies
  • Blog/writing front and center
Key Features:
  • Author bio with personal touch
  • "Currently learning" section
  • Blog post previews integrated
  • More text, story-driven
Pros
  • Very personal and authentic
  • Shows thought leadership
  • Engaging for readers
  • Flexible for future content
Cons
  • Requires blog content
  • Less technical/professional
  • More maintenance

Layout C: Balanced Professional

Clean modern design, safe middle ground (current improved)

Visual Style

  • Modern card-based layout
  • Professional but not corporate
  • Balanced white space
  • Subtle animations and polish
  • Clean, scannable sections

Content Focus

  • Balance of projects and experience
  • Technical depth without jargon
  • Clear but not salesy
  • Professional portfolio feel
Key Features:
  • Three-column "What I Build" cards
  • Technology badges organized by category
  • Experience highlights in clean list
  • Philosophy/approach section
Pros
  • Appeals to all audiences
  • Professional but safe
  • Easy to maintain
  • Good balance of all goals
Cons
  • Less distinctive
  • May feel generic
  • Doesn't stand out

Implementation Recommendation

Layout C (Balanced Professional) is already implemented in the current skills.html with updated content.
Layout A (Developer Portfolio) would be the best choice for minimizing day-job concerns while maximizing appeal to AI companies.
Layout B (Personal Tech Blog) would work great if you want to emphasize writing and thought leadership.
